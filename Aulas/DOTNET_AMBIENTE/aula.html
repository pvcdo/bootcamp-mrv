.Net Framework, Core e 5

O .net framework só roda em windows. O framework veio primeiro, em 2002, para a construção de páginas web e sistemas desktop que rodavam apenas em windows. Porém com a disseminação dos sistemas em nuvem, que utilizam em sua maioria o sistema linux foi necessário recriar o .net, criando o core. Para fazer a compatibilidade entre o Framework e o Core foi criado o .net standard, que é um contrato onde estão estabelecidas as bibliotecas de classe base (bibliotecas de compilação padrão). Em 2020 surge o .net 5, sendo uma única ferramenta para todas funcionalidades.

Para mais podemos buscar no google "NET Core/5 vs NET Framework"

Framework e Runtime

Framework é uma caixa de ferramentas prontas que nós podemos utilizar do jeito que quisermos. Runtime é a conversão do código alto nível para o nível binário que é o mais baixo.

Podemos saber mais pesquisando "Visão geral do CLR(Common Language Runtime)"

CLI do .Net

É possível dar vários comandos de dotnet no cli, como

  • dotnet --version => qual versão eu estou usando
  • dotnet --help => mostra ajuda sobre comandos
  • dotnet --info => mostra informações sobre o dotnet que estou usando
  • dotnet new => mostra quais opções de novo projeto nós podemos criar
  • dotnet run => executa o nosso projeto
  • dotnet build => compila o projeto
