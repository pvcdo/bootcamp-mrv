Conhecendo a Programação Orientada a Objetos

Apresentando a programação orientada a objetos

É um paradigma de programação, uma técnica, uma maneira de fazer programação. Esse paradigma tenta aproximar o mundo real da programação, pois tenta abstrair o mundo real dentro do código.

    Os pilares são:
  • Abstração
  • Encapsulamento
  • Herança
  • Polimorfismo

Os pilares são independentes e harmônicos, com exceção do polimorfismo que depende de herança.

Os principais conceitos da POO são as classes e objetos. Uma classe é um molde e um objeto é algo criado a partir desse molde.

Tipos de paradigmas

Existem vários de paradigmas, dentre os quais destacam-se:

  • POO
  • Estruturada
  • Imperativa
  • Procedural
  • Orientada a eventos
  • Lógica

Abstração

Introdução

Representar um objeto do mundo real considerando apenas os atributos importantes para o projeto. Por exemplo, uma criança que está em fase escolar tem diversas características, mas para um sistema informático de orientação a objetos da sua escola, o importante é o número da matrícula dela, as suas notas, faltas, nome, sala, série, dentre outros que fazem sentido no contexto da escola.

Requisitos do projeto

O professor indicou fazer o download do SDK do .NET no site da Microsofte também o VSCode. Irei fazer essa instalação, mesmo tendo o visual studio e instalando as "cargas de trabalho" como indicado pela nossa querida colega Giovanna Ramos.

Prática

  1. criar pasta do raiz projeto
  2. abrir o cmd nesse diretório
  3. comando dotnet new sln --name ExemploPOO
    Esse comando cria uma "solution" com o nome ExemploPOO. Essa solution é um agrupamento para o projeto.
  4. criou um novo diretório dentro da pasta do projeto com o nome ExemploPOO
  5. entrou nesse novo diretório e criou o projeto de fato com o comando dotnet new console
  6. depois voltou à pasta raiz do projeto e deu o comando dotnet sln add ExemploPOO/ExemploPOO.csproj
    Esse comando adiciona na solution o arquivo csproj especificado.
  7. abriu a pasta do raiz do projeto no vscode
  8. no vscode ele instalou as extensões
    • c#
    • c# extentions
    • vscode-icons
  9. dentro da pasta ExemploPOO ele criou uma outra chamada Models
  10. dentro da pasta Models ele criou uma classe c# de nome Pessoa.
    A extensão c# extentions então cria o arquivo cs com o nome Pessoa e dentro dele uma série de códigos para uma classe também de nome Pessoa)
  11. na primeira linha do recém criado documento ele colocou o comando using System;
  12. criou dentro da classe Pessoa os atributos Nome e Idade, com os adjetivos public, string e os verbos (get;set;) como parâmetros.
  13. criou dentro da classe Pessoa um método Apresentar() com os adjetivos public e void. O método faz Console.WriteLine($"Olá, meu nome é {Nome} e tenho {Idade} anos");
    O símbolo de cifrão no início do comando indica uma interpolação entre uma string e os atributos da classe.
  14. ele apertou F1 e no buscador do vscode que se abriu ele pesquisou omnisharp select project, isso para dar os autocomplete do c#. Ele escolheu o sln entre os dois que apareceram
  15. em ExemploPOO/Program.cs ele colocou dentro do método Main o texto Pessoa e deu um ctrl . para importar os métodos do ExemploPOO.Models, depois continuou escrevendo ao final Pessoa p1 = new Pessoa();
  16. p1.Nome = "Bob"; p1.Idade = 20; p1.Apresentar();
  17. para executar o programa nós devemos abrir o cmd no nível do csproj do nosso projeto e darmos o comando dotnet run
