Introdução a API's e métodos HTTP

O que é API

Aplication Program Interface. É uma camada que esconde as regras e possibilita comunicação externa. Facilita muito a integração pois os programas tem que se comunicar com outras tecnologias e aplicações. Para consumir um serviço de uma API não é necessário usar a mesma linguagem de criação da API. Existem várias metodologias para criar uma API, como o SOA, Rest ou por meio de SDK. A maioria das API atuais utilizam JSON.

O que são HTTP Methods e Status Code

Rest é um conjunto de regras para consumir uma API e RESTFUL é uma API que utiliza REST. Como parte dessas regras está o padrão de utilização dos métodos http conforme a seguir:

  • Get expera a obtenção de dados;
  • Post para criar
  • Put para atualizar
  • Delete para remover

A aplicação pode até ser configurada para consumir a API de uma forma diferente, mas é um outro padrão, não é RESTFUL.

Status code

Geralmente status começando com 2 (200, 201,...) é sucesso, 4 (404, 403,...) é erro no cliente e 5 (500, 504,...) é erro no servidor.

Status comuns:

  • 200 - ok
  • 401 - unauthorized
  • 403 - forbidden
  • 404 - Not found
  • 500 -
  • 504 - Gateway Timeout
