Parte 1 - Introdução

  • O objetivo é fazer um CRUD de séries de televisão.

Revisão

Classes abstratas

São classes que podem conter métodos abstratos. Essas classes não são instanciadas, mas só servem como "base" para outras classes, porém, as classes que herdam seus métodos e atributos devem implementá-los. As subclasses herdam as classes (1 classe-filha => 1 classe-mãe).

Interfaces

Semelhante a classes abstratas, mas não possui atributos. Não pode definir como os métodos devem ser implementados. Basicamente é uma lista de métodos que devem ser implementados. As classes implementam interfaces (1 classe-filha => n interfaces).

    14. Parte 3

  14. O professor vai usar um design pattern que utiliza uma interface como repositório. No caso do projeto para armazenar as séries cadastradas no app. Para isso vai criar Interfaces/IRepositorio.cs
  15. IRepositorio será a seguinte 
                using System.Collections.Generic;

            namespace DIO.Series.Interfaces
            {
                public interface IRepositorio
                {
                    List Lista(); //método Lista retorna uma lista de items de classe T
                    T RetornaPorId(int id); //retorna um objeto de classe T        
                    void Insere(T serie);        
                    void Exclui(int id);        
                    void Atualiza(int id, T serie);
                    int ProximoId();
                }
            }
  16. Em IRepositorio usará System.Collections.Generic e o namespace é DIO.Series.Interfaces
  17. faz public interface IRepositorio<T>, onde o T é uma classe genérica (ou seja, o repositório pode receber uma classe qualquer como tipo)
  18. criou uma classe chamada SerieRepositorio.cs, que implementa a interface IRepositorio
  19. dentro de SerieRepositorio instanciamos um 
     List<Serie> listaSerie = new List<Serie>
    e depois os métodos de IRepositorio

    No caso, listaSerie é inicialmente uma lista vazia que receberá items que são instâncias de Serie.cs

    20. Parte 4

  20. o método SerieRepositorio.Exclui apenas executa o método Serie.setExcluido que altera o atributo Serie.Excluido

    Isso serve para não apagar de fato uma informação, para não gerar reordenação dos índices da lista e outros problemas.
